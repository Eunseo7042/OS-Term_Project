# 개요
Producer-Consumer 문제는 운영체제의 대표적인 동기화 문제이다. 
요구에 따라 버퍼 크기가 0~30 범위에서 출력되도록 smaphore를 사용하여 문제를 해결하였다.

# 구현 방법
- pthread
- CV
- mutex

producer는 버퍼의 크기를 증가시키는 역할을 수행한다.
이때, X가 30 이상이면 대기하고, X가 30 미만일때 버퍼 크기를 증가시키도록 하여 버퍼의 크기를 요구하는 범위 내에서 출력할 수 있도록 하였다. 

while(x >= 30) { 
  pthread_cond_wait(&cond, &mutex); 
} 
x = x + 1;

consumer는 버퍼의 크기를 감소시키는역할을 수행한다.
이때, X가 0 이하이면 대기하고, X가 0 이상일 때 버퍼의 크기를 감소시키도록 하여 버퍼의 크기를 요구하는 범위 내에서 출력할 수 있도록 하였다. 

while(x <= 0) { 
  pthread_cond_wait(&cond, &mutex); 
} 
x = x - 1;

- 공유 변수 x에 접근하기 전에 mutux를 잠그고, 작업 후 해제하여 한 번에 하나의 스레드만 공유 변수에 접근할 수 있도록 할 수 있었고, 이를 통해 Race Condition을 제거하였다.

# 평가
Race Condition을 제거하면서 공유 변수의 일관성을 유지하였다.
버퍼 크기를 0~30 범위로 제한하여 요구사항을 만족하였다.
하지만, 스레드가 대기 상태에 들어갈 경우 성능저하가 발생할 수 있다.

# 결론 및 느낀 점
과제를 이해하는 것부터 쉽지 않았지만, 강의 자료를 찾아보며 힌트를 얻을 수 있었다. 
한학기 수업 중 많은 시간을 배운 동기화 문제를 직접 해결하는 과제를 통해 동기화 기법을 더 이해할 수 있었다.
